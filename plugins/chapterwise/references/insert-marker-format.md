# INSERT Marker Format

## Marker Structure

```html
<!-- INSERT
time: 2024-01-15T10:30:00Z
source: notepad
instruction: "after the hyperborean incursion"
confidence: 0.87
matched_after: "...the hyperborean forces retreated beyond the ridge."
-->
Elena drew her sword, the blade catching the firelight...
<!-- /INSERT -->
```

## Field Descriptions

- **time**: ISO 8601 UTC timestamp of when the insert was created
- **source**: Origin of the insert (e.g., "notepad", "cli", "clipboard")
- **instruction**: Original location instruction from user
- **confidence**: 0.00-1.00 confidence score for the matched location
- **matched_after**: Context text the insert was matched after

## Parsing Notes

Fields are parsed **order-independently**. The marker regex tolerates:
- Reordered fields
- Extra whitespace between fields
- Blank lines within the header
- Leading indentation (for markers inside YAML body blocks)

## Generation

Markers are generated by `insert_engine.py`'s `generate_insert_marker()` method.
The closing tag is always `<!-- /INSERT -->` on its own line.
